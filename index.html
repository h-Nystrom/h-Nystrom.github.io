Multiplayer Fishing

UE5 | Multiplayer | Blueprint | C++ | Gameplay Ability System | Niagara | 1 month

Fishing system perfect for multiplayer, co-op and single-player rpg games.
Created to be implemented easily into other projects that uses the gameplay ability system. It also stands on its own, with a nice northern Swedish inspired lake level with calming nature ambiance.

The inventory, equipment and item systems are mainly coded in C++ and then extended in blueprints, to interact with the rest of the game and make it easier for other developers to use. They were also built to utilize Epic games gameplay ability system to grant/remove and activate abilities depending on what item(s) the player has equipped. This makes it possible to completely encapsulate each ability away from the player character and make it much easier to add new types of items in the future.

I focused a lot on making the multiplayer experience as smooth as possible and how to handle the different kinds of events and interactions. Important events such as picking up or dropping items are handled authoritatively by the server to always validate the players inventory and equipment to avoid players from feeling cheated. While more action-oriented abilities are handled locally predicated to make them run a little faster but still checked by the server before it gets replicated to the other clients. Again, the gameplay ability system simplified this a lot by all it’s great replication options.
I also used the free plugin AGR-Pro for animation blending and replication.
And lastly, I used the advanced session plugin for hosting/joining servers via steam.
